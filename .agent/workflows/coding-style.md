# 클린 코드 표준 (Clean Code Standards)

**목표**: 간결하고, 명확하며, 스스로 설명되는 코드를 작성합니다.

## 핵심 원칙

1.  **SRP (단일 책임 원칙)**: 하나의 함수 = 하나의 역할.
2.  **KISS (단순성 유지)**: 과도한 엔지니어링 금지.
3.  **네이밍 (Naming)**: 의미 있는 이름 사용 (`user` 대신 `getUserById` 사용).
4.  **주석 금지 (No Comments)**: 코드는 주석을 지울 수 있도록 작성되어야 합니다. 설명이 필요하다면 리팩토링하십시오.

## 체크리스트

### 함수 (Functions)
- [ ] 20줄 이하인가?
- [ ] 오직 한 가지 일만 하는가?
- [ ] 인자가 최소화되었는가 (0-3개)?

### 구조 (Structure)
- [ ] **가드 절 (Guard Clauses)**: 중첩된 `if/else` 대신 조기 반환(early return)을 사용합니다.
- [ ] **상수 (Constants)**: 로직에 매직 넘버/문자열을 사용하지 않습니다.
- [ ] **배치 (Colocation)**: 관련된 코드는 서로 가깝게 배치합니다.

### 변경 사항 (Changes)
- [ ] 이 파일을 import하는 곳을 확인했는가?
- [ ] 의존성이 있는 파일들을 업데이트했는가?

> **좌우명**: "발견했을 때보다 코드를 더 깨끗하게 남겨라."
