# 간결한 계획 수립 (Concise Planning)

**목표**: 모든 코딩 작업에 대해 명확하고, 실행 가능하며, 원자적인 체크리스트를 생성합니다.

## 단계

### 1. 컨텍스트 스캔
*   `README.md`, `task.md`, 관련 코드 파일을 읽습니다.
*   제약 사항을 파악합니다 (예: "Streamlit 옵션만 사용해야 함").

### 2. 계획 생성
`implementation_plan.md`를 생성하거나 `task.md`를 업데이트합니다:

*   **접근 방식 (Approach)**: *무엇을* *왜* 하는지에 대한 1-3문장 설명.
*   **범위 (Scope)**: "포함(In)" 및 "제외(Out)" 항목을 글머리 기호로 작성.
*   **실행 항목 (Action Items)**: 6-10개의 원자적 단계 (동사로 시작).
    *   *예시*: "`app.py` import 리팩토링", "X 기능에 대한 단위 테스트 추가".
*   **검증 (Validation)**: 검증을 위한 구체적인 명령어 (예: `pytest tests/test_new_feature.py`).

### 3. 검토
*   단계가 원자적인가? (하나의 논리적 작업 단위)
*   파일과 모듈 이름이 구체적으로 명시되었는가?

> **템플릿**:
> ```markdown
> # 계획 (Plan)
> ## 접근 방식
> ...
> ## 실행 항목
> - [ ] 단계 1
> - [ ] 단계 2
> ## 검증
> ...
> ```
