# UI/UX 검토 체크리스트 (Review Checklist)

**목표**: "풍부한 미학(Rich Aesthetics)"과 "동적인 디자인(Dynamic Design)" 보장 (시스템 규칙 준수).

## 디자인 원칙
1.  **시각적 계층 구조 (Visual Hierarchy)**: 중요한 요소는 눈에 띄어야 합니다 (크기, 색상, 두께).
2.  **여백 (Whitespace)**: 요소들에게 여유를 주십시오. 혼잡함을 피하십시오.
3.  **피드백 (Feedback)**: 모든 클릭에는 반응이 있어야 합니다 (스피너, 토스트 알림, 전환 효과).
4.  **일관성 (Consistency)**: `styles.py`에 정의된 것과 동일한 색상/글꼴을 사용하십시오.

## 체크리스트 (감탄을 자아내는 요소)

### 미학 (Aesthetics)
- [ ] **현대적인 글꼴**: Inter/Roboto/Outfit을 사용하고 있습니까? (Times New Roman 금지).
- [ ] **색상 팔레트**: 조화롭습니까? (HSL, 세련된 다크 모드).
- [ ] **커스텀 룩**: 기본 스타일이 아닌, 프리미엄하고 "커스텀"된 느낌입니까?

### 상호작용 (Interaction)
- [ ] **호버 효과**: 버튼/카드가 호버 시 애니메이션됩니까?
- [ ] **전환 (Transitions)**: 페이지/탭 전환이 부드럽습니까?
- [ ] **로딩 상태**: 화면이 멈춰있지 않고 항상 스피너를 보여줍니까?

### 기능성 (Functionality)
- [ ] **반응형**: 모바일에서도 보기 좋습니까?
- [ ] **오류 처리**: 오류가 우아하게 표시됩니까 (스택 트레이스 노출 금지)?

> **규칙**: 지루해 보인다면, 그것은 버그입니다.
